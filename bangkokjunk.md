## Brand Context

Bangkok Junk is an English-language, Bangkok-focused junk removal brand. It sits within a portfolio of Biznitos-owned service sites (khunclean.com, naturalpestcontrolbkk.com, thaikru.com). The goal is to dominate search results for junk and item removal in Bangkok across multiple domains. Each domain targets distinct keyword clusters and audiences.

**Property types served:**
Approximately 75% condos and apartments, 25% houses, townhouses, and shophouses. Shophouse and house jobs tend to be larger and higher ticket. Language throughout the site should reflect all property types naturally ‚Äî avoid being condo-centric. Use "home," "property," or "space" as the default neutral term. Mention condos, houses, townhouses, and shophouses explicitly where relevant so all customers feel seen.

**Primary audiences:**
- **The Upgrader** ‚Äî replacing a sofa, fridge, mattress, or AC. Needs the old item gone fast. Coordinates around new delivery. Price-aware.
- **The Overwhelmed** ‚Äî accumulated junk, moving out, leaving Thailand, full clearout needed. Not price sensitive. Needs reassurance, not judgment.

## Translation System

Pages are automatically translated by the platform ‚Äî the builder does not need to write Thai content. However the builder must follow these rules precisely for the translation system to work:

**Rule 1 ‚Äî Use the `tr` class on every translatable text element.**
Any text that should be translated must have `class="tr"` on its element. Elements without this class will not be translated.

```html
<!-- Correct -->
<h1 class="tr">Junk Removal in Bangkok</h1>
<p class="tr">We pick up old furniture and appliances across Bangkok.</p>

<!-- Wrong ‚Äî will not be translated -->
<h1>Junk Removal in Bangkok</h1>
```

**Rule 2 ‚Äî No HTML or special symbols inside `tr` elements.**
Text inside any element with `class="tr"` must be plain text only. No nested HTML tags, no icons, no emoji, no special characters, no `<strong>`, `<em>`, `<a>`, `<br>`, or any other markup inside the element.

```html
<!-- Correct -->
<p class="tr">Fast, affordable junk removal in Bangkok.</p>

<!-- Wrong ‚Äî nested HTML inside tr element will break translation -->
<p class="tr">Fast, <strong>affordable</strong> junk removal in Bangkok.</p>
<p class="tr">Call us at <a href="tel:+66827973702">082-797-3702</a></p>
```

**Rule 3 ‚Äî Split mixed content into separate elements.**
If a block contains both translatable text and non-translatable elements (like a phone number, link, or icon), split them into separate elements. Only wrap the plain text portions with `class="tr"`.

```html
<!-- Correct -->
<p class="tr">Call us anytime</p>
<a href="tel:+66827973702">082-797-3702</a>

<!-- Wrong -->
<p class="tr">Call us anytime at <a href="tel:+66827973702">082-797-3702</a></p>
```

**Rule 4 ‚Äî All user-facing text must have the `tr` class.**
This includes: headings, paragraphs, card titles, card descriptions, button labels, FAQ questions and answers, form labels, placeholder text, footer text, navigation items, and meta content where applicable.

**Rule 5 ‚Äî Do not add `tr` to non-text elements.**
Do not add `class="tr"` to images, icons, links (only the link text inside), phone numbers, LINE IDs, email addresses, or URLs.

- Each entry must include: name, address, phone or LINE, hours, and 2-3 sentences of genuine description

- Write in a helpful, neutral tone ‚Äî do not promote Bangkok Junk within the list itself (a single contextual CTA at the end is fine)

**Internal linking rule:** Every resource list must link naturally to at least one relevant service page. Example: the mattress shops list links to /mattress-removal. The storage facilities list links to /condo-clearout. This is how resource content feeds service page authority.
